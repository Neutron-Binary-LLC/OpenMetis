# OpenMetis Agentic Workspace

This repository uses an agentic workflow to manage the development of neural-symbolic integration and financial mathematics. Multiple specialized AI agents collaborate to design, implement, and verify the OpenMetis framework.

## Core Agents

| Agent | Role | Focus |
|-------|------|-------|
| **ContextIngestor** | Researcher | Maps neural-symbolic patterns and financial math requirements. |
| **CodeGenerator** | Engineer | Implements neural blocks, math experts, and symbolic logic. |
| **MathAgent** | Mathematician | Numerical stability, symbolic correctness, and model accuracy. |
| **VisualizationAgent** | Analyst | Visualizing latent states, financial reasoning, and workspace evolution. |
| **ProjectScaffolder** | Architect | Maintaining repository structure and OpenMetis standards. |
| **Reviewer** | Quality Gate | Code quality, mathematical integrity, and performance review. |
| **TestGenerator** | QA | Automated unit, integration, and differentiability testing. |
| **HealthcheckAgent** | Monitor | Workspace integrity and documentation consistency. |

## Workflow

The typical development cycle follows this multi-agent pipeline:

1. **Discovery**: `ContextIngestor` analyzes requirements and maps existing code.
2. **Design**: `MathAgent` and `ProjectScaffolder` define the mathematical approach and structure.
3. **Implementation**: `CodeGenerator` implements the logic in `nn/` or `world/`.
4. **Verification**: `Reviewer` and `TestGenerator` validate changes for correctness and stability.
5. **Insights**: `VisualizationAgent` provides visual traces and latent state analysis.
6. **Maintenance**: `HealthcheckAgent` ensures the workspace remains consistent and functional.
